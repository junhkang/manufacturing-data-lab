# 🧭 Project 1: 제조 데이터 거버넌스 문서화 실습

본 실습은 제조 현장에서 자주 발생하는 기준 부재 문제를 해결하기 위한 **데이터 거버넌스 문서화 실습**입니다. 실습자는 실제 공정 데이터를 관리하는 오너의 역할을 가정하고, 기준 문서를 직접 설계하고 문서화합니다.

---

## 🎯 실습 목표

- 제조 데이터를 신뢰 가능한 형태로 만들기 위한 **기준 수립 및 문서화**
- Wiki.js 또는 AWS WorkDocs를 활용한 실무형 문서화 경험
- 후속 품질 진단 및 데이터 자동화 실습과 연계될 수 있는 기초 작업 수행

---

## 🧩 실습 시나리오

- 공정별 설비 코드, 품질 단위, 측정 기준이 제각각이라 자동화/리포트화가 어려운 상황
- 데이터 오너로서, 공정 데이터를 **표준화하고 기준을 명문화**해야 함
- 이후 품질 진단, ETL, 자동화 등의 과정에서 신뢰 가능한 데이터를 확보할 기반을 마련

---

## 🔧 실습 흐름

1. [`scenario.md`](./scenario.md) 실습 배경 읽기
2. [`preparation.md`](./preparation.md) 실습 준비사항 확인
3. `00_common/templates/`의 기준 문서 템플릿 확인
4. 선택한 도구 (Wiki.js / WorkDocs) 환경 준비
5. 기준 문서 작성 및 저장 (`wiki_output/` or `workdocs_output/`)
6. 작성 문서 제출 또는 다음 실습(품질 진단)으로 이동
